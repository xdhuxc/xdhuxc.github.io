SpringBoot 应用启动报错，Unregistering JMX-exposed beans on shutdown

2018-10-25
技术
约 510 字 预计阅读 2 分钟 次阅读

文章目录

本篇文章介绍了使用 Spring Boot 过程中遇到的一个错误及最终的解决方法，控制台输出的主要错误信息为：

1

AnnotationMBeanExporter : Unregistering JMX-exposed beans on shutdown

			<artifactId>spring-boot-starter-data-mongodb</artifactId>

使用搜索引擎搜索 Unregistering JMX-exposed beans on shutdown， 提出的方案基本都是显式添加 tomcat 的依赖，向 pom.xml 文件中加入如下内容：

1
2
3
4

<dependency>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-tomcat</artifactId>
</dependency>

但是很遗憾，并没有起多大作用，还是报同样的错误。 最后，尝试着修改了一下 Spring Boot 的版本，降级试试，于是，修改 pom.xml 文件为：

1
2
3
4
5

<parent>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-parent</artifactId>
	<version>1.5.12.RELEASE</version>
</parent>

终于可以正常启动 Spring Boot 程序了！

这应该是 Spring Boot 1.5.12.RELEASE 的一个bug。
