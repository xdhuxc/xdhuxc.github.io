Docker 合并多条 COPY 命令

创建于 2019-07-03 | 修改于 2018-07-03 | 总字数 259 | 阅读时长 1 分钟 | 阅读量 次
2019-07-03
Docker / Dockerfile

在编写 Dockerfile 时，我们可能需要将多条指令合并成一行，这样语法简洁，并且可以减少镜像层数。

将多条 COPY 指令合并为一条

比如，我们的 Dockerfile 里面有这样的语句：

FROM python:2.7-stretch
WORKDIR /dingtalk/
COPY dingtalk_callback.py /dingtalk/
COPY requirements.txt /dingtalk/
COPY settings.py /dingtalk/
RUN pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple
ENTRYPOINT ["python", "-u", "/dingtalk/dingtalk_callback.py"]

我们想把多个 COPY 指令合并成一条，可以改写该 Dockerfile 为如下形式：

FROM python:2.7-stretch
WORKDIR /dingtalk/
COPY dingtalk_callback.py requirements.txt settings.py /dingtalk/
RUN pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple
ENTRYPOINT ["python", "-u", "/dingtalk/dingtalk_callback.py"]

或者

FROM python:2.7-stretch
WORKDIR /dingtalk/
COPY ["dingtalk_callback.py", "requirements.txt", "settings.py", "/dingtalk/"]
RUN pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple
ENTRYPOINT ["python", "-u", "/dingtalk/dingtalk_callback.py"]

路径中包含空格时，必须使用这种形式。

COPY 指令支持通配符，如果复制多个文件时，文件名称本身有规律，则可以使用通配符来实现

FROM python:2.7-stretch
WORKDIR /dingtalk/
COPY dingtalk* /dingtalk/ # 复制所有以 "dingtalk" 开头的文件到 /dingtalk/ 目录下
RUN pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple
ENTRYPOINT ["python", "-u", "/dingtalk/dingtalk_callback.py"]
